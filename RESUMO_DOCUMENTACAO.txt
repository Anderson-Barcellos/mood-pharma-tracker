SCHEMA INDEXEDDB (Versão 3)
-----------------------------------------------------------------------------

medications:
- id (PK), name, category, halfLife, volumeOfDistribution
- bioavailability, absorptionRate, therapeuticRange
- createdAt, updatedAt
- Índices: name, category, createdAt, updatedAt

doses:
- id (PK), medicationId (FK), timestamp, doseAmount
- route, notes, createdAt
- Índices: medicationId, timestamp, [medicationId+timestamp]

moodEntries:
- id (PK), timestamp, moodScore, anxietyLevel
- energyLevel, focusLevel, sensitivityLevel, motivationLevel
- notes, createdAt
- Índices: timestamp, createdAt, moodScore

cognitiveTests:
- id (PK), timestamp, matrices, totalScore
- averageResponseTime, accuracy, createdAt
- Índices: timestamp, createdAt, totalScore

metadata:
- key (PK único), value, updatedAt
- Armazena flags de migração e timestamps
